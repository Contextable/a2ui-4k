# A2UI v0.8 Protocol Compliance

This document describes a2ui-4k's implementation status relative to the [A2UI v0.8 specification](https://github.com/google/A2UI).

> *The A2UI protocol is under active development. a2ui-4k tracks the specification and may update as the protocol evolves.*

## Server-to-Client Messages

| Message Type | Status | Notes |
|--------------|--------|-------|
| `beginRendering` | Supported | Initializes surface with root and optional styles |
| `surfaceUpdate` | Supported | Adds/updates components |
| `dataModelUpdate` | Supported | Updates data at specified paths |
| `deleteSurface` | Supported | Removes surface state |

See [State Management](../core-concepts/state-management.md) for usage details.

## Client-to-Server Messages

| Message Type | Status | Notes |
|--------------|--------|-------|
| `userAction` | Supported | Emitted as `UserActionEvent` |
| `dataChange` | Supported | Emitted as `DataChangeEvent` |

See [Events](../core-concepts/events.md) for usage details.

## Known Limitations

1. **Video/AudioPlayer:** These components render placeholders. Full media playback requires platform-specific implementation.

2. **Styles:** The `styles` object in `beginRendering` is stored but not currently applied to rendering.

3. **a2uiClientCapabilities:** Not yet implemented. This is used for catalog negotiation in A2A transport.

4. **JSONL Streaming:** a2ui-4k provides `SurfaceStateManager` for processing operations, but does not include JSONL stream parsing. Applications should implement their own transport layer.
